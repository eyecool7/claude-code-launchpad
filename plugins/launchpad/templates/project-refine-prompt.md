# 프로젝트 계획서 고도화 방법
이 문서는 사용자가 Claude Code Plan Mode에서 바로 사용 가능한 '프로젝트 계획서 고도화 프롬프트'를 제공하고 사용 방법을 설명한다.

## 사용법
1. project-refine-prompt.md를 열어서 프롬프트 내용을 claude code 채팅창에 붙여넣습니다.
2. claude code와 대화하며 project-plan.md 계획서를 고도화하세요.
3. 수정된 계획서를 프로젝트 루트에 project-plan.md로 다시 저장합니다. (기존 파일 덮어쓰기)
4. `/clear` 후 `/launchpad:setup` 명령어를 실행하여 다음 단계로 넘어갑니다.
   - refine 대화 히스토리가 setup 컨텍스트를 오염시키므로 반드시 `/clear`로 정리.

## 전제 조건
- 계획서 작성이 완료되어 `project-plan.md`가 루트에 있어야 한다.

# 프로젝트 계획서 고도화 프롬프트 (아래 전체를 Claude Code에 전달)

```
@project-plan.md 를 읽고, 아래 세 단계를 **모두** 완료해야 한다. 어떤 단계도 건너뛸 수 없다.
특히 2단계(스킬 & MCP 검색)는 반드시 실행해야 한다 — 인터뷰만 하고 끝내면 안 된다.

## 1단계: 구현 준비도 검증 (Gap-Driven Interview)

plan 단계에서 "뭘 만드는가"는 정의됐다. refine의 목표는 "이대로 코딩 시작할 수 있는가"를 검증하는 것이다.

### 1-A: 갭 분석

계획서를 아래 7개 영역의 **판정 기준**으로 평가한다.
각 기준은 "이 정보가 없으면 setup/build에서 막힘"을 기준으로 설계됨.

| # | 영역 | 판정 기준 ("구현 가능"의 정의) |
|---|------|---------------------------|
| 1 | **데이터 흐름** | 핵심 데이터의 입력→변환→출력 형태가 필드명 수준으로 명시됨 (예: 슬라이드 JSON에 title, points, templateId 필드) |
| 2 | **사용자 플로우 완전성** | 각 기능의 사용자 시나리오가 **입력 → 처리 → 결과 확인 화면 → 후속 액션** 전체를 커버하며, 각 지점의 UI 상태(로딩/에러/성공/빈값)가 정의됨. 유사 기능 간 플로우 대칭성 확인 (예: A에 미리보기가 있으면 B에도 있어야 하는지) |
| 3 | **외부 서비스 계약** | 각 외부 API의 인증 방식·실패 시 대응·제한사항(rate limit 등)이 명시됨 |
| 4 | **기술 제약 & 호환성** | 비표준 기술 조합이 있다면 구체적 해결 방법이 기술됨 (없으면 자동 ✅) |
| 5 | **에러 복구 경로** | 핵심 플로우별 실패→복구→사용자 안내가 코드로 옮길 수 있을 만큼 구체적 |
| 6 | **출력물 사양** | 최종 산출물의 포맷·크기·품질·파일명 규칙이 렌더링/저장 코드 작성 가능 수준 |
| 7 | **Phase 실행 가능성** | 섹션 6의 각 Phase 항목이 "이것만 보고 코딩 시작" 가능 수준 (에이전트·스킬·파일 위치·완료 기준 포함). 항목 설명과 완료 기준의 정합성 확인 — 항목이 "X를 구현"이면 완료 기준에도 X 동작 검증이 포함되어야 함 |

**평가 후 사용자에게 결과를 마크다운 테이블로 보여준다:**

> 📋 구현 준비도 갭 분석
>
> | # | 영역 | 판정 | 근거 |
> |---|------|:----:|------|
> | 1 | 데이터 흐름 | ✅/⚠️ | (한 줄 근거) |
> | 2 | 사용자 판단 지점 | ✅/⚠️ | ... |
> | ... | ... | ... | ... |
>
> ✅ N개 영역 충분 / ⚠️ M개 영역 보강 필요

### 1-B: 타깃 인터뷰

⚠️ 영역에 대해서만 AskUserQuestionTool로 질문한다.

**질문 규칙:**
- ✅ 영역은 절대 질문하지 않는다 (plan에서 이미 충분한 것을 다시 묻지 않음)
- 질문 필터: "이걸 모르면 setup 또는 build에서 구체적으로 뭐가 막히는가?"
- 영역당 1–2개 질문, 한 라운드에 최대 3개 질문
- 뻔한 질문 금지: 계획서를 읽으면 답이 나오는 것은 묻지 않는다
- 사용자가 "모르겠다" / "알아서 해줘" → 합리적 기본값 제안 + 계획서에 `[추정]` 표기
- **최대 3라운드.** 답변 반영 후 재평가 → 새 ⚠️ 발견 시 추가 라운드 진행, 3라운드 안에 종료

### 1-C: 완료 판정

아래 조건 중 하나를 만족하면 인터뷰 종료:
- 7개 영역 전부 ✅
- 남은 ⚠️가 setup/build 진행에 영향 없는 것으로 판단됨 (판단 근거 명시)

**종료 시 사용자에게 요약 출력:**

> ✅ 구현 준비도 검증 완료: 7개 영역 중 N개 보강됨.
> 검색 단계(2단계)로 넘어갑니다.

## 1.5단계: 인터뷰 반영

인터뷰에서 나온 변경사항을 계획서에 반영한다. 검색 전에 계획서를 최신화해야 누락 없이 검색할 수 있다.

반영 대상:
- **섹션 3** (검색 키워드): 인터뷰에서 새로 언급된 라이브러리·서비스·도메인을 키워드 테이블에 추가
- **섹션 4** (구현 설계): 스택 변경, Tier 변경, 에이전트/스킬 추가·삭제 반영
- **섹션 5** (기술 결정): 새 외부 서비스 추가 시 → 섹션 3 키워드에도 동시 추가 (섹션 5 연동 규칙)

반영 후 project-plan.md를 저장한다.

## 2단계: 스킬 & MCP 검색

먼저, 계획서 섹션 5 "외부 서비스 목록"에 있는 서비스 중 섹션 3 키워드 테이블에 빠진 것이 있으면 자동으로 추가한다.
(예: 섹션 5에 Stripe가 있지만 섹션 3에 stripe 키워드가 없으면 → 키워드 테이블에 `| 결제 | stripe, payment | MCP |` 추가)

그 다음, `.claude/skills/skill-discovery/SKILL.md`의 검색 절차를 실행해서
계획서 섹션 3 "툴 & 워크플로우 설계"의 검색 키워드 테이블을 실제 검색 결과로 채워줘.

구체적으로:
1. vive-md 리포를 클론 (`git clone --depth 1 https://github.com/johunsang/vive-md.git /tmp/vive-md`)
2. 계획서의 검색 키워드로 스킬 카탈로그 + MCP 카탈로그 검색
3. 검색 결과 테이블을 사용자에게 보여주고 "스킬 N개 + MCP N개로 계획서에 반영할까요?" 형태로 확인받기
4. 확인된 결과를 계획서 섹션 3에 반영한다. 반영해야 할 산출물 3가지:
   - **스킬 검색 결과 테이블** — 기능별 검색 키워드 + 결과 + **신뢰도(🟢/🟡/🟠/🔴)** + 설치/직접구현 판정
   - **MCP 서버 테이블** — 서비스별 MCP 서버명 + 설정
   - **`확정_MCP` JSON 블록** — setup 단계에서 .mcp.json 자동 생성에 사용되는 구조화된 데이터
5. 세 가지 모두 계획서 섹션 3에 기록한다.

검색 소스·절차·규칙은 모두 skill-discovery SKILL.md에 정의되어 있으니 그대로 따른다.

## 3단계: 섹션 6 갱신

1단계 인터뷰 + 2단계 검색으로 섹션 3, 4, 5가 업데이트되었다.
이제 **섹션 6(구현 순서)을 섹션 1–5와 다시 정렬**한다.

섹션 6의 각 Phase에서 다음을 확인하고 보강한다:
- **담당 에이전트** — 섹션 4 에이전트 구조에서 매핑 (Tier 1이면 생략)
- **활용 스킬** — 섹션 4 스킬 목록 + 섹션 3 검색 결과에서 매핑 (해당 없으면 생략)
- **파일 위치** — 섹션 4 폴더 구조에서 (주요 파일만)
- **외부 서비스** — 섹션 5에서 매핑 (해당 없으면 생략)
- **완료 기준** — 섹션 2 단계별 검증과 일치하는지 확인

인터뷰에서 기능 추가/삭제, Tier 변경, 에이전트 구조 변경이 있었다면 Phase 구성 자체도 조정한다.

## 최종: 완료 체크리스트

project-plan.md를 저장하기 **전에** 아래 체크리스트를 반드시 확인한다.
하나라도 미완이면 해당 단계로 돌아가서 완료한 후 저장한다.

- [ ] 1단계 구현 준비도 검증 완료 (7개 영역 전부 ✅ 또는 잔여 ⚠️ 영향 없음 판정)
- [ ] 1.5단계 인터뷰 반영 → project-plan.md 저장 완료
- [ ] 2단계 vive-md 클론 + 스킬 카탈로그 검색 실행
- [ ] 2단계 MCP 카탈로그 검색 실행
- [ ] 섹션 3에 "(검색 필요)" 또는 빈 결과가 남아있지 않음
- [ ] 섹션 3에 스킬 테이블 + MCP 테이블 + 확정_MCP JSON 블록 기록 완료
- [ ] 검색 결과를 사용자에게 보여주고 확인받음
- [ ] 3단계 섹션 6 갱신 — 각 Phase에 에이전트·스킬·파일 위치·외부 서비스 매핑 완료

**CRITICAL: 2단계를 건너뛰면 안 된다.** 커뮤니티에 검증된 스킬이 있는데 직접 만드는 건 낭비다.
인터뷰만 하고 끝내지 말 것 — 검색까지 완료해야 refine이 끝난 것이다.
**CRITICAL: 3단계를 건너뛰면 안 된다.** 섹션 4에 에이전트·스킬을 정의하고 섹션 6에 반영 안 하면 구축 시 활용되지 않는다.

모두 완료됐으면 최종 스펙을 project-plan.md에 저장하고, 사용자에게 다음 안내를 출력해줘:

---
✅ refine 완료. 다음 단계로 넘어가려면:
1. `/clear` 명령어로 현재 대화를 정리하세요 (refine 히스토리가 setup 컨텍스트를 오염시키므로 필수).
2. 새 세션에서 `/launchpad:setup` 을 실행하세요.
---
```
